Entendendo a Comunicação Cliente x Server.

(Índice)

O Cliente é o que consome, utiliza a Internet.

O Servidor é o que hospeda os Sites.

A Internet são os protocolos HTTP.

O Cliente acessa um Site pela Internet e a informação retorna do Servidor para o Cliente em forma de HTML, CSS, Javascript e Imagens. Salvando através de memória Cache, o Servidor grava estas informações no computador do Cliente. Acaso o utilizar o Site novamente, já está gravado em Cache, otimizando o tráfego de informações. O Cliente pode acessar os sites através de vários dispositivos existentes, Smartfones, Tablets, PC's, Notebooks entre outros dispositivos móveis. Vale lembrar que podemos ter resultados diferentes ao utilizar Navegadores diferentes num mesmo dispositivo, mesmo com a padronização dos navegadores e suas compatibilidades.

Navegadores.

(Índice)

Os Navegadores estão do lado do Cliente são chamados de Browser's, estes interpretam os arquivos que vem dos Servidores até o Navegador dos Clientes. Para haver compatibilidade entre os Navegadores ou Browser's estes seguem o padrão W3C.

Aplicações Web.

(Índice)

O Servidor acessa o Banco de Dados usando um Login e Senha e utiliza a Internet como meio de comunicação entre o Cliente e Servidor sem ter a necessidade de instalação. Quando o acesso se faz através de Navegador consideramos uma Aplicação Web e quando o acesso é feito através de um Aplicativo não consideramos uma Aplicação Web.

Dispositivos Móveis.

(Índice)

São considerados muito importantes! A cada dez Latinos-Americanos, nove possuem celulares e também 22%, passam mais de 22 horas em acesso nos celulares utilizando Twiter, Spotfy, Linkedin, Waze entre outros! Portanto concluímos que: "Trabalhar na Internet já não é o Futuro, já estamos Trabalhando nele!"

O que são Server's?

(Índice)

São os Servidores de Rede Local que atende à um grupo de Computadores (Intranet), já os Servidores conectados a Internet são Computadores com maior capacidade para guardar um número maior de arquivos, servindo outros Computadores. Existem muitos tipos de Servidores: Os Servidores de Arquivos onde se encontram Imagens, Vídeos e arquivos em geral. Os Servidores de Segurança são os que contém Firewall's. Os Servidores de Streaming são dados para consulta do usuário através de vídeos. Os Servidores de Email's onde se armazena, gerencia os Email's. Os Servidores Web armazenam arquivos para Internet, podem ser simples servidores ou enormes servidores que são chamados de Data Center's.

Hardware e Software em Servidores.

(Índice)

O Hardware é a parte eletrônica, seus periféricos e acessórios, placa mãe, CPU, monitor, memória RAM, discos rigidos, SSD's, gabinete, mouse, teclado. O Software é a parte de programas no computador, onde temos o Sistema Operacional, programas utilizados pelo usuário para execução de seus trabalhos. Existem vários Sistemas Operacionais o Windows Server para Servidores, Linux Ubuntu, Linux Debian, onde todos estes Sistemas Operacionais são Sistema UNIX. Quando solicitamos algo para o Servidor Web, o Sistema Operacional executa um programa que fará o processamento, podemos ter como exemplo o Apache e o NGINX. Ao fazer uma solicitação HTTP ao Servidor, o Sistema Operacional, solicita ao NGINX processar a informação HTTP e responder em HTML, CSS, Javascript e Informações de Texto.

Tipos de Servidores.

(Índice)

Temos vários tipos de Servidores, de Arquivo, de Segurança Firewall's, Streaming transmitem dados de áudio e vídeo pela internet em tempo real e continuamente. Servidores de Email armazena e gerência Email's, Servidores Web computadores que armazena arquivos que compõem o site, documentos, HTML, imagens, arquivos javascript, respondendo ao usuário. A Intranet é uma Servidor Web sem Internet, é um site dentro de uma Rede Corporativa, uma Rede Local. Servidor Proxy serve ao cliente com acesso restrito a Internet bloqueando sites não permitidos, tanto na solicitação quanto na resposta. Servidor Data Server equipamentos para armazenar e gerenciar dados de forma estruturada do Cliente e Servidor. Servidor Domain Name Service possui uma Lista de IP's,esse Servidor (DNS) lê o nome HTTPS e verifica na Lista qual é o Número de IP e direciona o IP ao seu domínio, por exemplo: https://meusite.com.br. O "br" é de Brasil!

Hospedagem de Sites.

(Índice)

É subir arquivos de Sites, disponibilizando na Internet para acesso dos Clientes. Para tanto usamos um Servidor Contratado e existem várias empresas de Hospedagem de Sites que oferecem Multiplataforma, ou seja, oferece vários tipos de Sistemas Operacionais, Registros de Domínio, Certificado SSL que dispõe uma conexão segura ("https" que é todo Criptografado!). Após criarmos um endereço de Domínio, vamos subir o Site através de uma conexão FTP (File Transfer Protocol), usando por exemplo o aplicativo para armazenar no Servidor Contratado, utilizando Usuário e Senha.

Linguagens de Programação.

(Índice)

São Linguagens escritas e formais especificando instruções e regras usadas para gerar os programas. As cinco Linguagens mais populares são: Javascript, Python, Java, PHP e C#. A Linguagem de Programação é uma Ferramenta para se chegar a uma soluição. Dependendo do uso, uma Linguagem poderá se destacar mais do que a outra.

Linguagem Server-side.

(Índice)

Linguagem do lado do Servidor e são elas: PHP, Ruby, Javascript, C#, Java, onde cada uma possúi suas próprias características. É importante aprender uma Linguagem de programção na qual está envolvida na Comunidade.

Linguagem Client-side.

(Índice)

Linguagen que roda no lado do Cliente, onde o Javascript executa aplicações no Navegador rodando HTML, CSS e Javascript que roda no lado cliente e também servidor. No GoogleChrome temos um interpretador que é o V8 engine, que também executa o Javascript. O Node.js é uma plataforma de desenvolvimento gratuita e de código aberto no qual podemos rodar Javascript estando fora de um Navegador. O Node.js também tem o interpretador V8 engine rodando o javascript no lado Cliente e no lado Servidor.

HTML não é uma Linguagem de Programação!

(Índice)

HTML é um arquivo de Marcação, uma Linguagem de Marcação de Hipertexto, que e um texto que aparece de diferentes modos no Navegador. Por Exemplo:

  • ID; Nome; Idade; Email.
  • 1; Maria Lucia; 43; maria@dio.me
  • 2; Jorge Almeida; 23; jorge@dio.me
Exemplo em HTML:
  • Eu curto demais a DIO!

Introdução ao HTML na Prática.

(Índice)

Vamos criar uma página HTML, para entender o funcionamento do Inspetor de Elementos. Vamos instalar o Visual Studio Code, e nele instalar vários Plugins que são recursos de controle de versões e controle de versões de códigos. Instalando o Live Server no VS Code, para isso, clicar em Extensions, depois em Search e digitar Live Server e depois instalar. Em seguida vamos criar um arquivo com o nome index.html. No VS Code já temos um Plugin chamado EMMET serve para ajudar na criação do código! Vamos abrir o arquivo criado e digitar html:5, enter, gerando um padrão pé-definido de código, auxiliando no desenvolvimento do código.

O Inspetor de Elementos.

(Índice)

É uma ferramenta de Desenvolvedor, no Navegador, acima e à direita, clicar no três pontinhos, depois em mais ferramentas, ferramentas do desenvolvedor, onde será aberto um console. No console, vamos clicar em Select an Element... e selecionar um elemento qualquer da página do Navegador, será mostrado no lado do console o HTML do item selecionado na página. No console passamos o mouse sobre os itens até acharmos o elemento escolhido na página. Depois podemos fazer uma alteração na cor do elemento escolhido, por exemplo.

Estrutura Básica do HTML.

(Índice)

Vamos criar uma pasta de projetos para o VS Code onde será feita toda a estrutura, criando um arquivo com o nome index.html.

O index é o início padrão das pastas!

Um arquivo HTML possui três partes:

  • As tags <html> abrir < / html> fechar.
  • Dentro do html temos <head> abrir cabeçalho < / head> fechar cabeçalho.
    • Tudo o que for escrito entre as tags <head> e < / head> são dados que serão carregados antes de serem exibidos para o usuário.
    • Dentro da tag <head>, temos a tag <title>Meu Primeiro HTML< / title>.
  • Também dentro do html temos <body> abrir corpo do texto da página < / body> fechar corpo do texto da página.
    • Tudo o que for escrito entre as tags <body> e < / body> aparece ao usuário.
Vamos clicar no nome do arquivo com o botão direito do mouse, no menu clicar em Open with Live Server, aparecerá um Navegador com a nossa página e o título do HTML. Toda vez que alterarmos ou salvarmos a nossa página atualiza automáticamente pelo plugin Live Server.

Falando sobre Tags.

(Índice)

Entre as tags <body>, podemos usar a tag < i > Significa texto em Itálico < / i > a tag <strong> Coloca o texto em Negrito < / strong>.

A tag <strong> mais a tag < i > Texto em Itálico e em Negrito < / i > < / strong>.

Existem tags que não tem fechamento como esta <input type="text"> que abre um campo para digitar algo no Navegador.

Atributos das Tags

(Índice)

São as propriedades das tags <strong id="Titulo"/ > "id" é um identificador do elemento no qual se quer alterar.

<strong style="Titulo"/ > coloca comandos do CSS direto no elemento.

<strong style="Color": blue/ > coloca cores.

<strong class="Titulo-principal"/ > onde o Título principal fica pré-definido.

<input type="text"/ > gera um campo para ser preenchido do tipo texto.

<input type="number"/ > gera um campo para ser preenchido do tipo número.

<input type="color"/ > gera um campo do tipo color com seletor de cores.

<img src="colocar o endereço da imagem aqui!"/ > src = "search"; endereço da imagem <img width="150" src="endereço da imagem"/ >.

Textos

(Índice)

Há vários "HTML's" voltados para textos:

  • As tags "H":
    • h1 - Título
    • h2 até h6 - Subtitulos
    • <p > Abre o Parágrafo < / p > Fecha o Parágrafo, é o espaço entre o Título e o Subtítulo, vem padronizado no Navegador e pode ser alterado no CSS.

Utilizando o Selecionador de Elementos do console do Desenvolvedor e passando o mouse sobre o texto este fica azul e as margens laranja, estes parâmetros que podem ser alterados no CSS.

Temos mais alguns tipos de tags.

  • A tag </ bloque cote> serve para fazer uma citação.
  • A tag <strong> Texto em Negrito < / strong>.
  • A tag <strong> mais a tag < u > Texto em Negrito e Sublinhado < / u > < / strong >.
  • A tag < i > Texto em Itálico < / i >.
  • A tag <mark> Texto em marca texto. < / mark >
  • A tag <sup> Apresenta um Asterisco < / sup > Apresenta um texto supra escrito, acima da linha normal e numa fonte menor, usado em fórmulas matemáticas, potenciação.
  • A tag <sub> Apresenta o texto subscrito, ou seja abaixo da linha normal numa fonte menor, usado para fórmulas químicas.
  • A tag <small> esta tag formata para textos menores, e pode ser melhorada no CSS.
  • A tag <font> especifica tipo, cor e tamanho da fonte do texto, usada até o HTML4.
  • A tag <del> esta tag define um texto que foi excluído.
  • A tag <abbr> apresenta a abreviação, por exemplo: HTML, CSS, ASSP - (O mais depressa possível!)

    • Dica!: Procurar no Google: w3schools tag html - para conhecer mais tipos de tags.

Listas Ordenadas e Não Ordenadas.

(Índice)

Nesta tag < ol > Abre uma Lista Ordenada e a tag < / ol > Fecha uma Lista Ordenada.

Dentro da tag < ol > temos a tag < li > que abre texto da Lista Ordenada e < / li > fecha o texto da Lista Ordenada.

Posso ter dentro da Lista Ordenada várias outras Listas Ordenadas.

Nesta tag < ul > abre uma Lista Não Ordenada e a tag < / ul > fecha uma Lista Não Ordenada.

Dentro da tag < ul > temos a tag < li > que abre texto da Lista Não Ordenada e < / li > fecha o texto da Lista Não Ordenada.

Também posso ter dentro da Lista Não Ordenada várias outras Listas Não Ordenadas.

Na frente da Lista Não Ordenada teremos Marcadores (Bolinhas) ao invés de uma numeração como na Lista Ordenada.

Links.

(Índice)

Navegando na Interner com a tag < a > Abrir Ancor e < / a > Fechar Ancor.

Por exemplo < a href="http//dio.me" > onde o "href" carrega o endereço do site.

O target serve para abrir uma nova aba, dessa forma ..._blank <target="_blank"> dio.me < / a >.

Se usarmos dessa forma, fica na mesma página ..._blank <target="_self"> dio.me < / a >.

Como uma forma de atalho, se clicarmos no site mais "Ctrl", abre o site numa outra aba.

Também ao clicar no site mais "Shift" conseguimos abrir outro Navegador.

É importante quando é seu site, abrir sempre na mesma aba para manter o usuário dentro do seu site!

Uma tag que sempre será usada é a < hr / > que traça uma linha abaixo do Título para destacá-lo, sendo que esta tag não possui conteúdo.
